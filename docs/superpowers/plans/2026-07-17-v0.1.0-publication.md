# GoAgents v0.1.0 公开发布实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `github.com/eruca/goagents` 添加 Apache-2.0 许可证，完成最终真实验收，并按依赖层发布 12 个 Go 子目录模块的 `v0.1.0`。

**Architecture:** 所有模块 tag 指向同一个最终发布 commit，但按依赖层分批推送。每批 tag 推送后都从仓库外的空临时目录、在 `GOWORK=off` 且无本地 `replace` 的条件下下载并编译代表模块；任一门禁失败立即停止后续发布。

**Tech Stack:** Git、GitHub、GitHub Actions、Go 1.26.1、Bash、OpenAI-compatible Qwen Provider、macOS Keychain。

## Global Constraints

- canonical public repository 必须是 `github.com/eruca/goagents`，SSH remote 必须是 `git@github.com:eruca/goagents.git`。
- 根许可证必须是 Apache License 2.0 官方文本，SPDX 标识为 `Apache-2.0`。
- 不得创建根 `v0.1.0`；只允许本文列出的 12 个子目录 tag。
- 12 个 tag 必须全部指向同一个通过本机最终验收和首次真实 CI 的 release commit；发布后的记录 commit 可以更新 `main`，但不得移动既有 tag。
- Provider endpoint、model、API key 和 token 不得写入仓库、Git 历史、命令输出或发布记录。
- 真实 Provider/Host 门禁出现 `SKIP` 不计为通过，必须停止发布。
- GitHub Actions 首次真实运行必须成功，才允许推送任何 tag。
- clean-consumer 必须在仓库外临时目录运行，使用 `GOWORK=off`，且不得使用本地 `replace`。
- 任一层 tag 或 clean-consumer 失败，立即停止，不发布下一层。
